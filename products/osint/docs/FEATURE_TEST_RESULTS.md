## Data-quality findings surfaced by the test
1. **`byline` is noisy** — often desk/outlet names ("Updated", "News Desk Follow on Twitter") → journalist-level scoring needs name normalization (today it's ~outlet-level).
2. **Stance extraction concentrates early** — wk1 116 signals / wk2 68 / wk3 3 / wk4 9. Time-series features (trajectory, who-flipped, gauge) thin out late-window; production needs even extraction coverage.
3. **`article_quotes.context` is sparse** but **`quote_text_en` is already backfilled** → translation is free; context-restore needs more coverage.
4. **Co-mention proxy** creates false feud pairs (Rahul→Sonia) and noisy co-mentions → a `target_entity_id` backfill on stances + `entity_type` filtering sharpen ~6 features at once.
5. **LLM mirrors source language** (Telugu in, Telugu out) → pin output language per user.
